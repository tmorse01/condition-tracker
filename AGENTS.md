## Working Style

Implement only the requested task scope.

Before making changes:

1. Summarize intended changes
2. List files you plan to modify

Constraints:

- Do not refactor unrelated code
- Do not rename files unless required
- Prefer existing patterns over introducing new abstractions
- Keep changes minimal and focused
- Avoid broad repo exploration unless necessary

Retry policy:

- Maximum 2 implementation retries
- If blocked, stop and summarize:
  - root cause
  - attempted fixes
  - recommended next step

Output style:

- Be concise
- Do not paste full files unless requested
- Show only important diffs and decisions

Definition of done:

- Relevant tests pass
- Type checks/lint pass if applicable
- Acceptance criteria satisfied
- No unrelated modifications
