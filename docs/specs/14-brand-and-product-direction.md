# Brand and Product Direction

# Working Product Positioning

A lightweight construction-loan document workflow platform.

Focus areas:

- borrower uploads
- internal review
- condition tracking
- document versioning
- modern UX

This should feel more modern than:

- Box folder workflows
- shared drives
- email attachment chains

---

# Product Personality

The product should feel:

```txt
Fast
Clean
Professional
Workflow-focused
Modern
Operational
```

Avoid:

```txt
Corporate banking UI
Overly playful startup branding
Heavy enterprise visual clutter
```

# UI Feel

Use:

- spacious layouts
- clean cards
- subtle borders
- modern status badges
- workflow-oriented screens

Avoid:

- dense tables everywhere
- old enterprise styling
- excessive gradients
- glassmorphism

# UX Priorities

The demo should communicate:

```txt
This feels like a real internal operational platform
```

even if simplified underneath.

---

# Demo Story

The product narrative:

```txt
A lender needs borrowers to upload required construction-loan documents.

Borrowers receive secure upload links.

Internal users review documents, request corrections, and track conditions.

The system keeps full version history and audit tracking.
```

This architecture could later evolve into:

- lender workflows
- approval routing
- document previewing
- construction draw workflows
